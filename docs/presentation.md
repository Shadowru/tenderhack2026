## Слайд 1: Проблема (20 сек)

**537 000 товарных единиц. 4 190 организаций. Одна поисковая строка.**

Заказчик больницы ищет «бумага» — ему нужна бумага для ЭКГ.
Заказчик школы ищет «бумага» — ему нужны бланки для грамот.
Оба получают одинаковую выдачу: SvetoCopy A4.

**Наша задача**: каждый видит то, что нужно именно ему.

---

## Слайд 2: Решение — 3 уровня интеллекта (40 сек)

### Уровень 1: Морфологически точный поиск
- Русская морфология: «карандаши» = «карандаш»
- **Subject extraction**: из «тачка с ручкой» выделяем предмет «тачка», а не «ручка»
- Поиск «ручка» → ручки. Не тачки. Не двери с ручками.

### Уровень 2: Опечатки и синонимы
- «Карндаш» → *Возможно, вы имели в виду: Карандаш*
- «прнтер» → *Принтер*
- Авто-коррекция раскладки: `,evfuf` → «бумага»
- Словарь синонимов: ПК = персональный компьютер = ЭВМ

### Уровень 3: Персонализация
- История 2 млн контрактов → профиль каждой из 4 190 организаций
- Cold start: при первом визите профиль уже готов
- Динамическая адаптация: каждый клик меняет следующую выдачу

## Слайд 6: Ключевые инновации (20 сек)

1. **Subject extraction** через морфологический анализ — выделяем ПРЕДМЕТ товара, а не атрибуты. Поиск «ручка» не находит «тачку с ручкой».

2. **Двухрежимный поиск**: короткие запросы (< 6 символов) → prefix по subject. Длинные → fuzzy multi_match. «Руч» → ручки, не ручные инструменты.

3. **Cold start из реальных контрактов**: 4 190 организаций получают персонализацию с первого визита на основе 2 млн исторических закупок.

4. **Персонализация выпадающих подсказок**: даже автокомплит учитывает профиль — школа видит учебники, больница видит лекарства.
