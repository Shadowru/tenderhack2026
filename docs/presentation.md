## Команда | TenderHack 2026

## Слайд 1: Проблема (20 сек)

**537 000 товарных единиц. 4 190 организаций. Одна поисковая строка.**

Заказчик больницы ищет «бумага» — ему нужна бумага для ЭКГ.
Заказчик школы ищет «бумага» — ему нужны бланки для грамот.
Оба получают одинаковую выдачу: SvetoCopy A4.

**Наша задача**: каждый видит то, что нужно именно ему.

---

## Слайд 2: Решение — 4 уровня интеллекта (40 сек)

### Уровень 1: Морфологически точный поиск
- Русская морфология: «карандаши» = «карандаш»
- **Subject extraction через pymorphy3**: из «тачка с ручкой» выделяем «тачка», а не «ручка»
- Phrase match: «туалетная бумага» находит именно её, а не офисную

### Уровень 2: Опечатки и синонимы
- «Карндаш» → *Возможно, вы имели в виду: Карандаш* (словарь 490K терминов)
- Авто-коррекция раскладки: `,evfuf` → «бумага»
- Синонимы: ПК = персональный компьютер = ЭВМ

### Уровень 3: Персонализация
- 2 млн контрактов → профиль каждой из 4 190 организаций
- Cold start: при первом визите профиль уже готов
- Каждый клик меняет следующую выдачу (dwell time tracking)

### Уровень 4: AI-расширение (RAG)
- Нечёткий запрос → LLM (Qwen 2.5 7B) выбирает категории из реального каталога
- «Медицинские перчатки» → находит нитриловые, хирургические, латексные

### Демо 1: Персонализация

   - Столовая + расходники — профиль колледжа

3. **Больница** → «бумага»
   - SvetoCopy Classic A4 — стандарт для медучреждения
   - Баннер: *[Показать без персонализации]* — сравните сами

### Демо 2: Опечатки + Subject

4. **«Карндаш»** → Карандаш (не клей-карандаш!)

## Слайд 5: Архитектура (20 сек)

```
Пользователь → Caddy (HTTPS) → Nginx → React SPA
                                      ↓
                               FastAPI Backend
                            ↙    ↓      ↓     ↘
                   Elasticsearch Redis  PG    Ollama
                   537K docs   profiles 2M    Qwen 2.5 7B
                   BM25+phrase boosts   buyer  AI expansion
                   synonyms    cache    history RAG-поиск
                   subject     events   metrics
```

Docker Compose, 7 сервисов + SonarQube. Всё на одном сервере (32 ядра, 125GB RAM).

## Слайд 6: Ключевые инновации (20 сек)

1. **Subject extraction** — морфоанализ pymorphy3 выделяет ПРЕДМЕТ товара. Падежи, сокращения, предлоги — всё учтено.

2. **Трёхстратегийный поиск**: prefix (< 6 символов) + fuzzy multi_match + phrase match. Каждый тип запроса обработан оптимально.

3. **Cold start из 2M контрактов**: 4 190 организаций с готовыми профилями из реальных закупок. Персонализация работает с первого визита.

4. **RAG через Ollama**: LLM + реальные категории из индекса. «Медицинские перчатки» → нитриловые, хирургические, латексные.

5. **93% качество** на 134 тестовых запросах. SonarQube Quality Gate: OK.
